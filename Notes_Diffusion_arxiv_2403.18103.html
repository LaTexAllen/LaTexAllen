I. Langevin Dynamics

The Langevin dynamics for sampling from a known distribution p(x)p(\bold{x})p(x) is an iterative procedure for t=1,…,Tt = 1,\dots , Tt=1,…,T :

xt+1=xt+τ∇xlog⁡p(xt)+2τz,z ∼N(0,I),\bold{x}_{t+1} = \bold{x}_t + \tau \nabla_\bold{x} \log p(\bold{x}_t) +\sqrt{2\tau} \bold{z}, \bold{z}~\sim \mathcal{N}(0,\bold I),xt+1​=xt​+τ∇x​logp(xt​)+2τ​z,z ∼N(0,I),

where τ\tauτ is the step size which users can control, and x0\bold{x}_0x0​ is white noise, lim⁡t→Txt=p(x)\lim\limits_{t \rightarrow T}\bold{x}_t = p(\bold{x})t→Tlim​xt​=p(x).

Without 2τz\sqrt{2\tau} \bold{z}2τ​z (the noise term), the equation is gradient descent. With 2τz\sqrt{2\tau} \bold{z}2τ​z, the equation is stochastic gradient descent.

II. (Stein’s) Score Function

The Stein’s score function is the gradient wrt the data point x\bold{x}x.
sθ(x)≔∇xlog⁡pθ(x).\bold{s}_{\bm{\theta}}(\bold{x}) \coloneqq \nabla_\bold{x} \log p_{\bm{\theta}}(\bold{x}).sθ​(x):=∇x​logpθ​(x).

Do not confuse it with the ordinary score function sx(θ)≔∇xlog⁡pθ(x)\bold{s_x}(\bm{\theta}) \coloneqq \nabla_\bold{x} \log p_{\bm{\theta}}(\bold{x})sx​(θ):=∇x​logpθ​(x) (the gradient of log-likelihood wrt θ\bm{\theta}θ). Maximum likelihood estimation uses the ordinary score function, whereas Langevin dynamics uses Stein’s score function.

∇xlog⁡pθ(x)=a vector field=[∂log⁡p(x)∂x,∂log⁡p(x)∂y]T.\nabla_\bold{x} \log p_{\bm{\theta}}(\bold{x}) = \textit{a vector field} = [\frac{\partial \log p(\bold{x})}{\partial x}, \frac{\partial \log p(\bold{x})}{\partial y}]^T.∇x​logpθ​(x)=a vector field=[∂x∂logp(x)​,∂y∂logp(x)​]T.

The data points drift in the direction of the vector field.

III. Score Matching Techniques

Explicit Score Matching
Given a dataset χ=x1,…,xM\chi=\bold{x}_1, \dots, \bold{x}_Mχ=x1​,…,xM​, we consider the kernel density estimation by defining a distribution

Since q(x)q(\bold{x})q(x) is an approximation to p(x)p(\bold{x})p(x) which is never accessible, we can learn sθ(x)\bold{s}_{\bm{\theta}}(\bold{x})sθ​(x) based on q(x)q(\bold{x})q(x). This leads to the following definition of the a loss function which can be used to train a network.

II. Forward and Backward Iterations in SDE

If we introduce a noise term zt ∼N(0,I)\bold{z}_t~\sim \mathcal{N}(0,\bold I)zt​ ∼N(0,I) to the gradient descent algorithm, then the ODE will become a stochastic differential equation (SDE).
